Attio has shipped three updates to its developer platform. Records and list entries can now be filtered by a saved view — developers can pass a view_id into any record or entry query to apply its filters without manually reconstructing filter JSON. A new list views endpoint lets developers retrieve a view_id for any object or list.

The release also introduces three new MCP tools: updating list entries by list entry ID, updating list entries by record ID, and prepending, appending, or renaming notes via an update-note action. Full details are available in Attio's developer documentation.
